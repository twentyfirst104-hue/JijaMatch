# 🍬 Telegram-бот «Каталог вкусов для вейпа» (свайп-формат)

Бот-каталог вкусов жидкостей для вейпа и одноразок в формате Tinder: пользователь
листает карточки (👍 лайк / 👎 дизлайк / ⭐ в избранное), а встроенный ИИ через
OpenRouter тегирует вкусы, анализирует анкету и пишет **персональные описания вкуса**.

---

## ⚠️ ВАЖНЫЕ ПРЕДУПРЕЖДЕНИЯ (прочитайте обязательно)

### 1. Юридический риск и блокировки
- **Через бота ничего нельзя купить.** Это только каталог вкусов и описаний.
  Никаких цен, заказов и намёков на покупку — так задумано намеренно.
- Telegram **может заблокировать бота** за вейп/никотиновую тематику. Это риск,
  который нельзя устранить полностью.
- **Поэтому критично делать регулярные бэкапы** (бот делает автобэкап раз в сутки
  и шлёт файл вам в личку; также есть команда `/backup`). Храните копии БД у себя.
- При регистрации пользователь обязан подтвердить совершеннолетие (18+) и видит
  дисклеймер.

### 2. Лимиты бесплатного OpenRouter (ИИ)
Бесплатный тариф OpenRouter имеет ограничения:
- **50 запросов в день** на бесплатных моделях;
- **либо 1000 запросов в день**, если вы один раз пополните баланс на **$10**
  (эти деньги **не сгорают** — лежат на счёте, лимит просто поднимается);
- **20 запросов в минуту**.

👉 **Настоятельно рекомендуем пополнить OpenRouter на $10.** Простыми словами: вы
платите 10 долларов один раз, деньги остаются вашими, но дневной лимит ИИ
вырастает с 50 до 1000 запросов — этого хватит для нормальной работы бота.

Бесплатные модели нестабильны и иногда «выпадают», поэтому в боте реализована
**fallback-цепочка**: если основная модель не ответила — автоматически
пробуются запасные. Если ИИ совсем недоступен — вкус сохраняется без тегов и
описания, а позже его можно догенерировать командой `/retag`.

## 🚀 Локальный запуск

### Шаг 1. Получите ключи
1. **Токен бота** — напишите [@BotFather](https://t.me/BotFather) → `/newbot` → скопируйте токен.
2. **Свой Telegram ID** — напишите [@userinfobot](https://t.me/userinfobot), он покажет ваш числовой ID. Это будет админ.
3. **Ключ OpenRouter** — зарегистрируйтесь на https://openrouter.ai, создайте ключ на https://openrouter.ai/keys (рекомендуем пополнить на $10).

### Шаг 2. Настройте `.env`
Скопируйте пример и впишите свои значения:

**Windows (PowerShell или cmd):**
```bat
copy .env.example .env
notepad .env
```

**macOS / Linux:**
```bash
cp .env.example .env
nano .env
```

Заполните как минимум: `BOT_TOKEN`, `ADMIN_ID`, `ADMIN_USERNAME`, `OPENROUTER_API_KEY`.

### Шаг 3. Создайте окружение и установите зависимости

**Windows:**
```bat
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**macOS / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Шаг 4. Запустите бота
```bash
python bot.py
```
Вы увидите в консоли «Бот запущен. Ожидаю сообщения…». Откройте бота в Telegram и
нажмите `/start`.

> Остановить бота: `Ctrl + C`.

## ☁️ Деплой на Amvera

Amvera — российский хостинг, удобно держать бота онлайн 24/7.

1. Зарегистрируйтесь на https://amvera.ru и создайте новый проект (тип — приложение).
2. Загрузите код проекта (через git push в репозиторий Amvera или загрузкой файлов).
   Файл `amvera.yml` уже настроен (Python 3.11, запуск `python bot.py`).
3. **Постоянное хранилище.** Чтобы база данных не терялась при перезапусках,
   Amvera монтирует постоянный диск в каталог `/data`. В **переменных окружения**
   проекта (раздел настроек Amvera) задайте:
   ```
   BOT_TOKEN=ваш_токен
   ADMIN_ID=ваш_id
   ADMIN_USERNAME=ваш_username
   OPENROUTER_API_KEY=ваш_ключ
   DB_PATH=/data/bot.db
   BACKUP_DIR=/data/backups
   ```
   (остальные переменные — по желанию, у них есть значения по умолчанию).
4. Запустите проект. В логах Amvera должно появиться «Бот запущен».

> Если предпочитаете Docker — в проекте есть `Dockerfile`. Тогда так же
> смонтируйте постоянный том в `/data` и задайте те же переменные окружения.

**Резервные копии:** бот раз в сутки присылает вам файл БД в личку. Дополнительно
скачивайте копии командой `/backup` и храните у себя — это защита на случай
блокировки бота.

## 🛠 Шпаргалка для админа

Откройте бота и используйте команды (доступны только вам, по вашему `ADMIN_ID`):

| Команда / кнопка | Что делает |
|---|---|
| `/admin` | Открыть админ-панель |
| **Добавить производителя** | Создать бренд (название, описание, ссылка на канал) |
| **Одобрить производителей** | Одобрить пользователей, попросившихся в производители (антиспам) |
| **Добавить линейку** | Производитель → категория → название → общее фото |
| **Добавить вкус** | Линейка → название → описание → фото → (для конструкторов: пропорция). После — авто ИИ-тегирование |
| **Все карточки** | Все линейки и вкусы (даже чужие/на модерации). Редактировать, перетегировать ИИ, удалять |
| **Заявки на модерацию** | Очередь pending; платные вверху с ⚡. Одобрить / отклонить с комментарием / редактировать и одобрить |
| **Управление производителями** | Включить платный статус на 30 дней, продлить, отключить, статистика |
| **Статистика** | Пользователи, топ лайков/дизлайков, карточки по статусам, платные производители |
| `/retag` | Догенерировать теги и базовые описания для вкусов, где их нет (после сбоев ИИ) |
| `/backup` | Получить файл БД себе в личку прямо сейчас |
| `/broadcast` | Рассылка всем пользователям (с подтверждением; заблокировавшие — помечаются) |
| `/cancel` | Отменить текущий мастер (добавление/рассылку) |

### Что даёт платный статус производителю
1. Ссылка на Telegram-канал в карточках (у бесплатных скрыта).
2. Приоритет в выдаче (мягкий бонус, не «накрутка»).
3. Ускоренная модерация (метка ⚡, вверху очереди).
4. Расширенная статистика (по каждому вкусу, конверсия, динамика 30 дней).
5. Отметка «✓ Официальный производитель».
6. Безлимит вкусов (у бесплатных лимит — переменная `FREE_PRODUCER_FLAVOR_LIMIT`, по умолчанию 20).

Оплата — **только напрямую вам по СБП**. Статус вы включаете вручную в
«Управление производителями». За 3 дня до окончания и в день окончания вам
придёт уведомление (по одному разу).

## ⚙️ Настройки алгоритма (в `.env`, можно не трогать)

| Переменная | Смысл | По умолч. |
|---|---|---|
| `LIKE_WEIGHT_COEFFICIENT` | Сила лайка | 1.0 |
| `FAVORITE_WEIGHT_COEFFICIENT` | Сила добавления в избранное (сильнее лайка) | 1.5 |
| `PAID_PRIORITY_COEFFICIENT` | Бонус платным в выдаче (1.3 = +30%) | 1.3 |
| `DISLIKE_RESET_THRESHOLD` | Насколько должен сместиться профиль, чтобы вернуть дизлайки в ленту | 15.0 |
| `FREE_PRODUCER_FLAVOR_LIMIT` | Лимит вкусов для бесплатного производителя | 20 |

## ❓ Частые вопросы

**ИИ не проставляет теги / описания пустые.**
Скорее всего, исчерпан лимит OpenRouter или не задан `OPENROUTER_API_KEY`.
Проверьте ключ и баланс, затем выполните `/retag`.

**Бот не запускается.**
Проверьте `.env`: должны быть заданы `BOT_TOKEN` и `ADMIN_ID`. Текст ошибки
бот печатает в консоль при старте.

**Где хранятся данные?**
В файле `data/bot.db` (локально) или `/data/bot.db` (на Amvera). Делайте бэкапы!
